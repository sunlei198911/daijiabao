# 全局说明

## url

示例:
```
POST: /api/login
```

前面是请求方式，有 GET、POST、PUT、Delete。后面是 url，数据请求全部以 `/api/` 为前缀。

## 入参

`username`  用户名
`password`  密码

## 出参

出参示例：
```json
{
    "code": 0,
    "message": "",
    "data": {
        
    }
}
```

出参说明：

`code` 0: 数据正常，1: 登录超时，2: 无权限访问，此处为全局状态设置，方便做跳出登录等统一处理
`message` 数据描述信息，可以为空字符串
`data` 业务数据部分，如登录是否成功在此处定义，可参见登录接口示例
